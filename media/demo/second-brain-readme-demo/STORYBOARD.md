# Second Brain README Demo — Storyboard

Format: 16:9, 1920×1080, silent motion-first README demo.

Target length: 56 seconds.

## Research-backed direction

- Keep the video short and problem-led: open with the pain, then show the product transformation and the workflow.
- Make it understandable without sound because README previews and linked assets are often watched muted.
- Use a clickable thumbnail in the README and host/link the MP4 asset rather than relying on raw Markdown video behavior.
- Build in Hyperframes as plain HTML/CSS with a paused GSAP timeline so the demo can be rendered deterministically.

## Story spine

### 0–5s — The problem

Message: “Knowledge is trapped in different mental maps.”

Visual: Nikolay, Goria, and Agent cards hold disconnected fragments. Lines are broken and floating.

Purpose: Start from the actual problem, not the feature list.

### 5–12s — Layer 1: shared knowledge

Message: “A governed shared knowledge layer.”

Visual: captured entries flow into a central knowledgebase with owner, visibility, source, citations, versions, and time.

Purpose: Explain the durable foundation.

### 12–22s — Layer 2: translation

Message: “The product is translation between mental maps.”

Visual: a GitHub repo discovery from Nikolay is translated into Goria’s data-quality context and agent-domain use cases.

Purpose: Show the real value: reusable knowledge in the recipient’s language.

### 22–32s — Layer 3: Hermes living organism

Message: “Hermes profiles keep the knowledgebase alive.”

Visual: Research Scout, Engineering Librarian, and Quality Critic run scheduled jobs and create digests/proposals.

Purpose: Show proactive domain agents without implying this repo hosts Hermes itself.

### 32–42s — Trust boundary

Message: “Agents propose. Humans approve. History stays intact.”

Visual: citations, proposals, audit log, bitemporal recall, and “no direct storage access” guardrails.

Purpose: Make autonomy feel safe, not magical.

### 42–52s — MCP workflow

Message: “Connect any MCP-compatible agent.”

Visual: Claude, ChatGPT, Codex, Cursor, and Hermes connect through the MCP/API boundary to Second Brain skills.

Purpose: Make the product feel immediately usable.

### 52–56s — Close

Message: “Second Brain — Living Team Knowledgebase.”

Visual: the three layers lock into one system.

Purpose: Leave a clear README-ready product promise.
